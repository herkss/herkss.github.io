### 변수(variable)
다음은 변수를 생성하고 초기화하는 예입니다.
```dart
var name = 'Bob';
```
변수는 참조를 저장합니다. 호출된 변수 에는 "Bob" 값을 가진 객체 name에 대한 참조가 포함됩니다 .String

변수 의 유형은 name로 추론되지만 String, 지정하여 해당 유형을 변경할 수 있습니다. 객체가 단일 유형으로 제한되지 않은 경우 유형을 지정 Object하거나 dynamic필요한 경우 지정합니다.
```dart
Object name = 'Bob';
```
또 다른 옵션은 추론될 유형을 명시적으로 선언하는 것입니다.
```dart
String name = 'Bob';
```

이 페이지는 지역 변수에 대해 유형 주석 대신 를 사용하는 것을 권장하는 스타일 가이드를 따릅니다.var

### null safety
#
Dart 언어는 안전한 null 안전성을 강화합니다.

Null 안전성은 로 설정된 변수의 의도치 않은 액세스로 인해 발생하는 오류를 방지합니다 null. 이 오류는 null 역참조 오류라고 합니다. null 역참조 오류는 로 평가되는 표현식에서 속성에 액세스하거나 메서드를 호출할 때 발생합니다 null. 이 규칙의 예외는 또는 null와 같이 속성이나 메서드를 지원하는 경우입니다 . null 안전성을 통해 Dart 컴파일러는 컴파일 시간에 이러한 잠재적 오류를 감지합니다.
```dart
toString()hashCode
```
int예를 들어, 변수 의 절대값을 찾고 싶다고 가정해 보겠습니다 i. 가 i이면 null호출하면 i.abs()null 역참조 오류가 발생합니다. 다른 언어에서는 이를 시도하면 런타임 오류가 발생할 수 있지만 Dart 컴파일러는 이러한 동작을 금지합니다. 따라서 Dart 앱은 런타임 오류를 일으킬 수 없습니다.

Null 안전성은 세 가지 주요 변경 사항을 도입합니다.

변수, 매개변수 또는 다른 관련 구성 요소에 대한 유형을 지정할 때 유형이 .을 허용하는지 여부를 제어할 수 있습니다 . Null 허용을 활성화하려면 유형 선언 끝에 . null을 추가합니다 .?
```dart
String? name  // Nullable type. Can be `null` or string.

String name   // Non-nullable type. Cannot be `null` but can be string.
```

변수를 사용하기 전에 초기화해야 합니다. Nullable 변수는 기본적으로 로 설정 null되므로 기본적으로 초기화됩니다. Dart는 Nullable이 아닌 유형에 초기값을 설정하지 않습니다. 초기값을 설정하도록 강제합니다. Dart는 초기화되지 않은 변수를 관찰하도록 허용하지 않습니다. 이렇게 하면 수신자의 유형이 될 수 있지만 사용된 메서드나 속성을 지원하지 않는 속성에 액세스하거나 메서드를 호출할 수 null없습니다 null.

nullable 유형이 있는 표현식에서는 속성에 액세스하거나 메서드를 호출할 수 없습니다. nulllike hashCode또는 를 지원하는 속성이나 메서드인 경우에도 동일한 예외가 적용됩니다 toString().

사운드 널 안전성은 잠재적인 런타임 오류를 편집 시간 분석 오류 로 변경합니다 . 널 안전성은 다음 중 하나일 때 널이 아닌 변수를 플래그합니다.

null이 아닌 값으로 초기화되지 않았습니다.
값 을 할당했습니다 null.
이 검사를 통해 앱을 배포하기 전에 이러한 오류를 수정할 수 있습니다 .

### 기본값
#
nullable 유형을 가진 초기화되지 않은 변수는 초기 값이 null.입니다. 숫자 유형의 변수조차도 처음에는 null입니다. 숫자는 Dart의 다른 모든 것과 마찬가지로 객체이기 때문입니다.

int? lineCount;
assert(lineCount == null);
콘텐츠_복사
메모
프로덕션 코드는 assert()호출을 무시합니다. 반면 개발 중에는 조건 이 거짓 이면 예외를 throw합니다 . 자세한 내용은 Assert 를 확인하세요 .assert(condition)

null 안전성을 사용하려면 사용하기 전에 null이 불가능한 변수의 값을 초기화해야 합니다.
```dart
int lineCount = 0;
```
선언된 곳에서 로컬 변수를 초기화할 필요는 없지만 사용하기 전에 값을 할당해야 합니다. 예를 들어, 다음 코드는 Dart가 lineCount전달될 때까지 null이 아닌 것을 감지할 수 있기 때문에 유효합니다 print().
```dart
int lineCount;

if (weLikeToCount) {
  lineCount = countLines();
} else {
  lineCount = 0;
}

print(lineCount);
```
최상위 수준 변수와 클래스 변수는 지연 초기화됩니다. 초기화 코드는 변수가 처음 사용될 때 실행됩니다.

### late
#
수정 late자에는 두 가지 사용 사례가 있습니다.

선언 후 초기화되는 null이 불가능한 변수를 선언합니다.
변수를 지연하여 초기화합니다.
Dart의 제어 흐름 분석은 종종 null이 아닌 변수가 사용되기 전에 null이 아닌 값으로 설정되는 것을 감지할 수 있지만, 때로는 분석이 실패합니다. 두 가지 일반적인 경우는 최상위 변수와 인스턴스 변수입니다. Dart는 종종 설정되었는지 확인할 수 없으므로 시도하지 않습니다.

변수가 사용되기 전에 설정되었다고 확신하나 Dart에서 설정되지 않는 경우 변수를 다음과 같이 표시하여 오류를 수정할 수 있습니다 late.
```dart
late String description;

void main() {
  description = 'Feijoada!';
  print(description);
}
```
알아채다
변수를 초기화하지 못하면 late해당 변수를 사용할 때 런타임 오류가 발생합니다.

변수를 late선언에서로 표시하고 초기화하면 변수가 처음 사용될 때 초기화자가 실행됩니다. 이 게으른 초기화는 몇 가지 경우에 유용합니다.

변수가 필요하지 않을 수도 있고, 변수를 초기화하는 데 비용이 많이 들 수도 있습니다.
인스턴스 변수를 초기화하고 있으며, 초기화 프로그램에서는 .에 액세스해야 합니다 this.
다음 예에서 temperature변수가 전혀 사용되지 않으면 비용이 많이 드는 readThermometer()함수는 호출되지 않습니다.
```dart
// This is the program's only call to readThermometer().
late String temperature = readThermometer(); // Lazily initialized.
```
### final 및 const
#
변수를 변경할 생각이 없다면, 타입 대신 final또는 를 사용 하거나 타입에 추가로 사용합니다. final 변수는 한 번만 설정할 수 있습니다. const 변수는 컴파일 타임 상수입니다. (Const 변수는 암묵적으로 final입니다.)constvar

메모
인스턴스 변수는 가능 final하지만 const. 은 불가능합니다.

변수 를 생성하고 설정하는 예는 다음과 같습니다 final.
```dart
final name = 'Bob'; // Without a type annotation
final String nickname = 'Bobby';
```
변수 의 값은 변경할 수 없습니다 final.

✗ 정적 분석: 실패
```dart
name = 'Alice'; // Error: a final variable can only be set once.
```
컴파일 타임 상수가const 되기를 원하는 변수에 사용합니다 . const 변수가 클래스 수준에 있는 경우 로 표시합니다 . 변수를 선언하는 곳에서 값을 숫자나 문자열 리터럴, const 변수 또는 상수 숫자에 대한 산술 연산 결과와 같은 컴파일 타임 상수로 설정합니다.static const
```dart
const bar = 1000000; // Unit of pressure (dynes/cm2)
const double atm = 1.01325 * bar; // Standard atmosphere
```
키워드 const는 상수 변수를 선언하기 위한 것만은 아닙니다. 상수 값을 만드는 데 사용할 수도 있고, 상수 값을 만드는 생성자를 선언하는 데 사용할 수도 있습니다. 모든 변수는 상수 값을 가질 수 있습니다.
```dart
var foo = const [];
final bar = const [];
const baz = []; // Equivalent to `const []`
```
위와 같이 선언 const의 초기화 표현식에서 생략할 수 있습니다 . 자세한 내용은 DON'T use const redundantly를 참조하세요 .constbaz

값이 있었던 경우에도 final이 아니고 const가 아닌 변수의 값을 변경할 수 있습니다 const.
```dart
foo = [1, 2, 3]; // Was const []
```
변수 의 값은 변경할 수 없습니다 const.

✗ 정적 분석: 실패
```dart
baz = [42]; // Error: Constant variables can't be assigned a value.
```
유형 검사 및 캐스트 ( is및 as), 컬렉션if 및 스프레드 연산자 ( ...및 ...?)를 사용하는 상수를 정의할 수 있습니다 .
```dart
const Object i = 3; // Where i is a const Object with an int value...
const list = [i as int]; // Use a typecast.
const map = {if (i is int) i: 'int'}; // Use is and collection if.
const set = {if (list is List<int>) ...list}; // ...and a spread.
```
메모
객체는 final수정할 수 없지만, 필드는 변경할 수 있습니다. 반면, const객체와 필드는 변경할 수 없습니다. 불변 입니다 .
